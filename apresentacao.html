CRUD de Barbearia em PHP para apresentacao de TCC

Este projeto foi desenvolvido para demonstrar um sistema simples e funcional de agendamento, com cadastro, edicao, exclusao, listagem e resumo de informacoes em uma unica aplicacao.

Slide 2

Objetivo do sistema

  • Registrar agendamentos de clientes da barbearia.
  • Permitir cadastrar, editar, excluir e listar atendimentos.
  • Exibir indicadores rapidos como total, confirmados e faturamento previsto.
  • Servir como exemplo simples, autoexplicativo e facil de demonstrar.

Problema resolvido

Em vez de controlar agenda manualmente, o sistema centraliza os dados do atendimento, do cliente, do barbeiro e do servico em uma interface unica.

Slide 3

Estrutura do codigo

index.php

Controla o fluxo principal: recebe requisicoes, valida formulario, chama o repositorio e renderiza a interface.

src/bootstrap.php

Cria a conexao com o banco SQLite, inicializa a tabela e define funcoes auxiliares.

src/AppointmentRepository.php

Centraliza as operacoes no banco: listar, buscar, criar, atualizar, excluir e resumir.

Slide 4

Fluxo de funcionamento

1

Entrada do usuario

O usuario acessa a tela principal e preenche o formulario de agendamento.

2

Tratamento dos dados

O `index.php` normaliza os campos e valida se nome, telefone, servico, horario e valor foram informados.

3

Persistencia

Os dados validados sao enviados ao `AppointmentRepository`, que executa SQL via PDO.

4

Retorno visual

O sistema recarrega a lista, mostra resumo atualizado e exibe mensagem de sucesso ou erro.

Slide 7

Repository: onde esta o CRUD

all() e find()

Leem os registros no banco e permitem listar ou buscar um item especifico.

create() e update()

Inserem e atualizam agendamentos com `prepare` e parametros nomeados.

delete() e summary()

Apagam registros e calculam os indicadores que aparecem no topo da tela.

Slide 8

Validacoes e seguranca basica

  • normalizacao dos dados antes de salvar
  • validacao dos campos obrigatorios
  • conversao de data e valor para formato consistente
  • uso de `htmlspecialchars` na saida com a funcao `e()`
  • uso de `PDO::prepare` para evitar SQL montado manualmente

Por que isso importa

Mesmo sendo um projeto didatico, ele ja aplica boas praticas basicas: validar entrada, escapar saida e separar acesso ao banco.

Slide 10

Banco de dados

  • O banco utilizado no projeto e o SQLite.
  • O arquivo fica salvo em `database/barbearia.sqlite`.
  • A tabela principal se chama `agendamentos`.
  • Os principais campos sao: `id`, `cliente`, `telefone`, `servico`, `barbeiro`, `data_horario`, `valor`, `status`, `observacoes` e `created_at`.
  • Na primeira execucao, a tabela ja e criada automaticamente pelo sistema.
CREATE TABLE IF NOT EXISTS agendamentos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cliente TEXT NOT NULL,
    telefone TEXT NOT NULL,
    servico TEXT NOT NULL,
    barbeiro TEXT NOT NULL,
    data_horario TEXT NOT NULL,
    valor REAL NOT NULL,
    status TEXT NOT NULL DEFAULT "Agendado",
    observacoes TEXT DEFAULT "",
    created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
)
